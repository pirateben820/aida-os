# Config Files

## config/models.yaml
Model registry. Loaded by `aidad/registry.py`.

Key fields per entry:
- `id` — must match the Ollama model name exactly (e.g. `qwen2.5-coder:7b`)
- `provider` — `ollama` | `openai` | `anthropic` | `vllm` | `groq` | `openrouter` | `gemini`
- `skills` — list from: `coding`, `reasoning`, `math`, `general`, `planning`, `vision`, `embedding`
- `requirements.vram_gb` — minimum VRAM needed (0 for CPU-only or cloud)
- `requirements.min_ram_gb` — minimum system RAM
- `cloud` — true means requires API key, never pulled locally
- `starter` — true means auto-pulled by bootstrap and seeded automatically

**Disk budget rule: total `starter` model sizes must stay under 12 GB.**

Current starter models: qwen2.5-coder:7b (4.7GB) + deepseek-r1:1.5b (1.1GB) + nomic-embed-text (0.3GB) = ~6.1GB

## config/agents.yaml
Capability definitions. Loaded by `capabilities/model.py`.

Structure:
```yaml
agents:
  <name>:
    description: "..."
    preferred_skills: [coding, general, ...]
    capabilities:
      can: [read_files, write_files, ...]
      cannot: [manage_secrets, execute_shell, ...]
```

Default is **deny** — unlisted capabilities are forbidden.

## config/settings.yaml
Runtime settings. Loaded by `aidad/settings.py` into a Pydantic `Settings` model.

Key settings:
- `daemon.port` — aidad API port (default 8421)
- `nats.url` — NATS connection string
- `ollama.url` — Ollama API base URL
- `privacy.air_gapped` — disables all cloud AI when true
- `privacy.default_cloud_policy` — `ask` | `never` | `allow`
- `local_first` — scheduler prefers local models when true

## config/model_torrents.yaml
Auto-generated by the seeder. Maps model IDs to magnet links.
Used by setup wizard to download models via P2P instead of Ollama.
Do not edit manually.
